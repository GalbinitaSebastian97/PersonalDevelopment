   mirrorFrameless_MirrorControl

.. uml::
    @startuml
    title "Frameless Mirror Fold Control"
    start
    
    :Read manual and auto adjustment commands;
    :Read current fold command;
    :Determine if request is new or ongoing;
    :Check abort conditions;
    
    if (Abort reason detected?) then (Yes)
      :Reset auto adjustment command;
      :Reset active fold command;
      :Set abort reason;
      :Write abort reason to output;
      stop
    else (No)
      if (New fold command received?) then (Yes)
        if (NVM not busy and position save allowed?) then (Yes)
          :Save current X and Y positions to NVM;
          :Disable position save flag;
          :Write NVM block;
        endif
        :Trigger fold movement;
        :Clear abort reason;
      else if (New unfold command received?) then (Yes)
        :Trigger unfold movement;
        :Clear abort reason;
      else (Ongoing movement)
        if (Auto adjustment active?) then (Yes)
          if (Auto adjustment finished?) then (Yes)
            :Set abort reason - Position reached;
            :Disable auto adjustment;
            :Clear active fold command;
          else if (Auto adjustment aborted?) then (Yes)
            :Set abort reason - Position unreachable;
            :Disable auto adjustment;
            :Clear active fold command;
          else (Still running)
            :Continue auto adjustment;
          endif
        endif
      endif
    endif
    
    if (Position save not allowed and\nmirror adjustment active?) then (Yes)
      if (NVM not busy?) then (Yes)
        :Enable position save flag;
        :Write NVM block;
      endif
    endif
    
    :Write abort reason to output;
    :Update previous fold command;
    
    stop
    @enduml

.. _SDD.FramelessFold.algorithms.30:

=================
Define Algorithms
=================

.. comment:: SDD.FramelessFold.algorithms.38

   In this module is implemented the frameless mirror fold control logic.
   The main algorithm is implemented in the function **mirrorFrameless_MirrorControl** which performs the following steps:
   1. Reads the current fold command request from the application.
   2. Determines if the request is new or ongoing based on the previous command.
   3. Checks for any abort conditions such as invalid position data, higher priority glass adjustment, block detection, or ongoing fold operation.
   4. If an abort condition is detected, it resets the fold and auto adjustment commands, sets the appropriate abort reason, and writes it to the output.
   5. If no abort condition is detected, it processes new fold or unfold commands by
      triggering the corresponding movements and managing the auto adjustment state.
   6. It also manages the position save to NvM before fold and updates the abort reason output and previous command state at the end of the function.
